知识截止日期训练数据的时间边界，直接影响模型对最新事件的认知官方说明时效性敏感场景必须关注
对齐训练方法SFT / RLHF / DPO / KTO / Constitutional AI / RLAIF 等技术报告影响安全性和输出风格
训练计算量 (FLOPs)预训练总算力消耗，衡量 Scaling Law 位置技术报告（部分不公开）算力 vs 性能的效率
1.3 上下文与模态标准上下文窗口默认支持的 context length（4K / 8K / 32K / 128K / 200K / 1M+）官方文档决定可处理的文本长度
最大扩展上下文通过技术手段可扩展到的最长上下文官方文档长文档/代码库场景
有效上下文长度实际性能不显著衰减的上下文范围（NIAH 测试）独立评测标称 vs 实际往往有差距
支持输入模态文本 / 图像 / 音频 / 视频 / 文件（PDF等）官方文档多模态场景覆盖
支持输出模态文本 / 图像生成 / 音频生成 / 结构化数据官方文档生成能力广度
1.4 开放性与许可开源 / 闭源 / 半开放完全开源（权重+代码+数据）/ 仅开放权重 / 完全闭源 API官方发布可定制性和数据安全

板块二：性能与效率

二级维度三级指标说明 / 对比要点测量方法对比意义
2.1 推理速度首 Token 延迟 (TTFT)从请求发出到收到第一个 token 的时间（ms），用户感知的"响应速度"API 测量 / 本地 benchmark交互体验的核心指标
输出速度 (TPS / Tokens per second)每秒生成的 token 数，影响长文本生成耗时API 测量批量处理和实时场景
端到端延迟 (E2E Latency)从请求到完整响应的总时间，含排队、计算、网络客户端计时综合体验
流式输出延迟Streaming 模式下 token 间的间隔是否均匀，有无卡顿流式 API 监测实时对话流畅度
Prefill 速度 vs Decode 速度输入处理速度（Prefill）和生成速度（Decode）的分别测量推理引擎日志长输入 vs 长输出场景取舍
2.2 吞吐与并发最大并发请求数API 同时处理的请求数上限 / Rate Limit (RPM/TPM)API 文档 / 压测高并发业务场景
批处理吞吐量单位时间内处理的总 token 数（Batch API 场景）Batch API 测试离线大规模处理
高负载下的性能衰减并发增加时延迟和质量是否显著下降压力测试生产环境稳定性
API 可用性 / SLA服务正常运行时间比例（99.9% / 99.99%），故障频率监控平台生产环境可靠性
2.3 资源消耗GPU 显存需求不同精度（FP16/INT8/INT4）下的推理显存占用推理引擎监测自托管硬件选型
推理能耗 (Watt/token)每生成一个 token 的功耗硬件监测绿色 AI 和运营成本
KV Cache 大小长上下文时 KV Cache 的显存占用增长推理引擎日志长上下文的实际可行性
2.4 长上下文性能性能随长度衰减曲线上下文从 4K→128K 时，各任务准确率的变化曲线NIAH / LongBench标称窗口 vs 有效窗口
大海捞针准确率 (NIAH)在不同深度和长度下定位关键信息的成功率NIAH 测试信息检索完整性
长文本推理质量长上下文下推理、摘要、分析的质量是否保持LongBench / L-Eval实际长文档处理效果
